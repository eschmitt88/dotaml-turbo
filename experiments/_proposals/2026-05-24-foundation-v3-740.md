# foundation-v3-740 — the actually-grounded foundation model

## Where this fits

After three experiments, the design is now grounded in evidence rather than literature guesses:

1. **multitask-740** lifted the ceiling 0.6477 → 0.6495 via multi-task supervision (gradient-density unlock).
2. **foundation-mvp-740** stacked 4 new components and broke training (val_auc=0.5058). The anticipated diagnostic fork (baseline ablation works at 0.6470) confirmed: design, not scale.
3. **foundation-component-isolation-740** attributed cleanly: UW-SO is the saboteur; PMAE with EMA-teacher is safe; (team, team) bias is helpful (+0.0023).

v3 keeps every safe component, drops the saboteur, fixes one long-pending oversight (duration regression), and finally extends to broader cross-patch data — which is what makes the patch token meaningful in the first place.

## What's new vs v2 (and why each thing)

- **Drop UW-SO entirely**, revert to multitask-740's hand-tuned α weights (1.0/0.15/0.3/0.1). UW-SO with our per-task initial-loss normalization fix STILL collapsed omega to items in v2 because items per-class BCE converges to ~0.08 within ~100 batches, so L_k_init captures a too-low baseline and the formula then over-weights items by ~30×. Hand-tuned α was multitask-740's anchor and worked.
- **Duration as regression** (`SmoothL1` on `log(seconds)`), 1 scalar output. The 8-bucket CE was carried forward from multitask-740 by the foundation-mvp subagent despite my proposal text specifying regression. Switching now: preserves ordinal info (bucket 3 vs 5 ≠ bucket 3 vs 8 in CE); cleaner downstream queries ("win | duration=t" via regression head); matches the existing KDA/GPM/HD regression heads structurally.
- **Patch token tested on broader corpus.** foundation-mvp's `foundation_no_patch_token` ablation was meaningless because the training window was single-patch (so patch_id was degenerate). v3 trains on Aug 2025 → Feb 2026 (~3 patches), so the patch token finally has multiple values to encode.
- **Data extension is the prerequisite.** This is the ~3-4h pre-build that foundation-mvp's subagent punted. Necessary to test the foundation hypothesis.

## What's KEPT from v2 (proven safe or helpful)

- **Canonical hero sort** at load time (no per-slot positional encoding).
- **(team, team) 2×2 attention bias** (the +0.0023 lift component).
- **PMAE auxiliary with EMA-teacher** (momentum=0.996, stop-gradient — the Bug A fix).
- **Shared decoder + task-as-token prompting** (Whisper-style).
- **FT-Transformer skeleton** with first-layer first-LN removed.
- **5M-param scale** (proven neutral via baseline_multitask_repro at 0.6470).
- **Live monitoring discipline** per `~/.claude/CLAUDE.md`.

## Three result forks

- **val_auc ≥ 0.6508 (CONFIRMED).** Foundation framing works. The architectural lift from (team, team) bias + the data scale-up from cross-patch corpus compound. Downstream-query interfaces become the next direction (item rec with net_worth conditioning, lineup-vs-lineup, hero-pair synergy).
- **val_auc in [0.6493, 0.6508) (FLAT vs iso_teambias).** Data scale-up alone didn't add a lift beyond what (team, team) bias already provided. Still useful: this is the substrate for the multi-query unlock that single-task models can't provide. Per-patch val_auc diagnostic will tell us if cross-patch generalization is the issue.
- **val_auc < 0.6493 (REGRESSION).** Cross-patch confound hurts more than the bias helps. Patch token alone insufficient. Investigation: per-patch val_auc + maybe move to FiLM conditioning in v4.

## Engineering plan

- **Data pre-build (3-4h CPU)**: extend `experiments/2026-05-19-upstream-data-cleanup-740/build_features.py` over the foundation training window. Same defensive checkpointing (snapshot-time clamp + numpy-routed pyarrow write + row-group-stats verification; NO full re-read). Same for `experiments/2026-05-20-rich-supervision-multitask-740/build_rich_cols.py`.
- **Reuse v2 code verbatim** for everything except: drop UW-SO from loss.py (or set `use_uwso=False`), switch duration head from CE to SmoothL1 regression in models.py + train.py, configure `foundation_v3` ablation flags in config.yaml.
- **Smoke** the new design at 50K rows × 1 epoch to verify duration regression works end-to-end.
- **Profile** 1 epoch on full extended data to confirm wall estimate.
- **Run full training** in background with `python -u` and live monitoring polls every 30-45 min.

This is the first foundation experiment that can actually fail informatively about the foundation hypothesis (rather than about implementation bugs).
